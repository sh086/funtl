    # 数据库Driver连接失败

    # 问题描述

    ​ 原先数据库是5.6版本的，新数据库是8.0版本的。现将CRM连接新申请的MySQL8.0版本数据库启动，在运行时提示如下错误信息：

    Loading class `com.mysql.jdbc.Driver'. This is deprecated. The new driver class is `com.mysql.cj.jdbc.Driver'. The driver is automatically registered via the SPI and manual loading of the driver class is generally unnecessary.
    1

    # 解决方案

    # 修改mysql驱动版本

    ​ 方案一：在pom中显示声明mysql驱动版本为8.0版本，然后重新运行

    # 在pom中显示声明mysql驱动版本为8.0版本
<dependency>
     <groupId>mysql</groupId>
     <artifactId>mysql-connector-java</artifactId>
     <version>8.0.11</version>
 </dependency>
 # 重新安装
 mavn clean install

    # 修改driverClassName

    ​ 方案二：该问题可能是因为升级后的mysql驱动类不同，所以，可以尝试在配置文件中修改driverClassName为com.mysql.cj.jdbc.Driver，参考这里 (opens new window)。

    spring:
  datasource:
    driverClassName: com.mysql.cj.jdbc.Driver
